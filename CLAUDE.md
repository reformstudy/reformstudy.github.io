# Claude Instructions for reformstudy.github.io

## Project Identity
**reformstudy.github.io** is a React + TypeScript web application for the ReformStudy platform—a digital study tool for biblical and theological research. The application is deployed as a GitHub Pages site with automated CI/CD via GitHub Actions.

## Technology Stack & Tooling
- **Runtime**: Node.js (ES modules)
- **Frontend**: React 18 with TypeScript 5
- **Bundler**: Vite 5 (with React plugin)
- **Icons**: lucide-react
- **Hosting**: GitHub Pages (source: `main` branch, `/docs` folder)
- **Type Checking**: TypeScript compiler (part of build)

## Development Workflow
### Initial Setup
```bash
npm install
```

### Local Development
```bash
npm run dev
```
Starts Vite dev server with HMR. In dev mode, resources are fetched from `/docs/resources/`.

### Building
```bash
npm run build
```
- Runs TypeScript type checking via `tsc`
- Bundles the React app with Vite into `docs/`

### Build Resources Only
```bash
npm run build:resources
```
Processes `res/` source files → `docs/resources/`. Run this when you add or change resource JSON files.

### Convert Raw Text to Resource JSON
```bash
npm run convert:resources -- --type <bible|confession|commentary|strongs> --input <file> --id <id> --name "Name"
```

### Preview
```bash
npm run preview
```
Serves the built `docs/` folder to verify production output.

## Resource System

The resource system is the core data layer. Resources (Bible, confessions, commentaries, Strong's concordances) are large JSON files served statically from `docs/resources/` and fetched on demand.

### How to Access Resources in Components
**Always use the `useResources()` hook** — do not import from `resourceLoader.ts` directly in components:

```typescript
import { useResources } from '../context/ResourceContext';

function MyComponent() {
  const { bibles, confessions, commentaries, strongsGreek, strongsHebrew, ensureResourceLoaded } = useResources();

  useEffect(() => {
    ensureResourceLoaded('kjv');          // Bible
    ensureResourceLoaded('wcf');          // Westminster Confession of Faith
    ensureResourceLoaded('matthew-gill'); // Commentary
    ensureResourceLoaded('strongs-greek');
    ensureResourceLoaded('strongs-hebrew');
  }, [ensureResourceLoaded]);

  const kjv = bibles['kjv']; // null until loaded
}
```

### ResourceContext API (`src/context/ResourceContext.tsx`)
```typescript
interface ResourceContextType {
  manifest: ResourceManifest | null;
  bibles: Record<string, BibleVersion>;       // keyed by id, e.g. 'kjv'
  confessions: Record<string, Confession>;    // keyed by id, e.g. 'wcf'
  commentaries: Record<string, Commentary>;   // keyed by id, e.g. 'matthew-gill'
  strongsGreek: StrongsConcordance | null;
  strongsHebrew: StrongsConcordance | null;
  isLoading: boolean;                         // true while manifest loads
  error: Error | null;
  loadedResources: Set<string>;
  ensureResourceLoaded: (resourceId: string) => Promise<void>;
}
```

The manifest is fetched on app init. Individual resources are lazy-loaded and cached in context state. `isLoading` reflects manifest initialization only — check that the specific resource (e.g. `bibles['kjv']`) is truthy before rendering content.

### Resource Types and IDs
| Type        | IDs available            | Context key       |
|-------------|--------------------------|-------------------|
| Bible       | `'kjv'`                  | `bibles`          |
| Confession  | `'wcf'`                  | `confessions`     |
| Commentary  | `'matthew-gill'`         | `commentaries`    |
| Strong's    | `'strongs-greek'`, `'strongs-hebrew'` | `strongsGreek`, `strongsHebrew` |

### Bible Book IDs
Bible books use abbreviated uppercase IDs (e.g. `'GEN'`, `'EXO'`, `'MAT'`, `'REV'`). Testament values are `'OT'` or `'NT'`. Example:
```typescript
const book = kjv.books.find(b => b.id === 'MAT');
const verses = book.verses.filter(v => v.chapter === 1);
```

For advanced or non-component usage (scripts, utilities), `resourceManager` from `resourceLoader.ts` provides `loadBible(id)`, `loadConfession(id)`, `getVerse()`, `getChapter()`, etc. See `RESOURCES.md` for full type definitions and data formats.

## Key Architectural Patterns
- **`App.tsx`** wraps everything in `<ResourceProvider>` then renders `<AppContent>` which manages active module state
- **`useResources()`** is the only way components should read resource data
- **`ensureResourceLoaded(id)`** is idempotent — safe to call multiple times; does nothing if already loaded
- **`AtlasAndTimeline`** contains hardcoded `eraEvents` data; several eras have empty `events: []` (stubs)
- **`GenericView`** is a placeholder stub — `archive` and `theology` modules need dedicated components

- **Build Output**: `docs/` (required for GitHub Pages)
- **React Plugin**: `@vitejs/plugin-react` with TSX support
- **Resource URL in dev**: `/docs/resources/` — in production: `/resources/`

## Code Standards & Patterns
1. **Components**: Functional components only, use TypeScript for all props
2. **File Naming**: PascalCase for `.tsx` files (e.g., `ScriptureReader.tsx`)
3. **Module System**: ES6 `import/export`
4. **Styling**: CSS custom properties from `styles.css`; component-level via inline styles or `className`
5. **Type Safety**: Define all prop types; avoid `any`
6. **Icons**: lucide-react only
7. **Resources**: Access via `useResources()` hook, not raw `resourceLoader.ts` imports

## Common Scenarios

### Adding a New Module (e.g. implementing `archive`)
1. Create `src/components/ConfessionalArchive.tsx` with TypeScript prop types
2. Use `useResources()` to access `confessions` and call `ensureResourceLoaded('wcf')`
3. In `App.tsx`, import the component and replace the `archive` case in `renderWorkspace()`
4. Add global styles to `styles.css` using existing CSS custom properties

### Accessing Bible Data
```typescript
const { bibles, ensureResourceLoaded } = useResources();
useEffect(() => { ensureResourceLoaded('kjv'); }, [ensureResourceLoaded]);
const kjv = bibles['kjv'];
if (!kjv) return <LoadingState />;
const book = kjv.books.find(b => b.id === 'GEN');
const chapter1 = book?.verses.filter(v => v.chapter === 1) ?? [];
```

### Adding a New Resource
1. Add the JSON file to the appropriate `res/` subdirectory following the format in `RESOURCES.md`
2. Run `npm run build:resources` to publish to `docs/resources/`
3. Add the resource ID to `ResourceContext.tsx` `ensureResourceLoaded` handler
4. Access it via `useResources()` in components

### Fixing Type Errors
- Run `npm run build` to see all TypeScript errors
- Check `tsconfig.json` for strict settings
- Ensure all prop types are explicitly defined
